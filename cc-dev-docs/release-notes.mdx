# CreatorCommerce Developer Platform

> Updates to the CreatorCommerce Developer Platform, including the Unified API, client SDKs, and developer tools.

<Tip>
  For release notes on other CreatorCommerce products, see the [CreatorCommerce Help Center](https://support.creatorcommerce.com).
</Tip>

#### [Date - e.g., January 15, 2025]

* **Feature Name**: Brief description of the feature or update. Learn more in our [documentation](/path/to/docs).

* **API Update**: Description of API changes, including any new endpoints, parameters, or response formats. See the [API reference](/api-reference/endpoint) for details.

* **Breaking Change**: Description of any breaking changes with migration instructions. Read more in our [migration guide](/path/to/migration-guide).

* **Deprecation Notice**: Description of deprecated features with timeline and replacement options. See [deprecation documentation](/path/to/deprecations).

* **Bug Fix**: Description of bug fixes and improvements.

#### [Date - e.g., December 20, 2024]

* **New Endpoint**: We've launched a new endpoint `/creators/{creatorId}/drops/{dropId}/cart` for generating checkout URLs. Learn more in our [Checkout documentation](/api-reference/checkout).

* **SDK Update**: We've released version 2.0 of the CreatorCommerce JavaScript SDK with improved error handling and TypeScript support. See the [SDK documentation](/sdk/javascript) for details.

* **Rate Limits**: We've increased rate limits for the Unified API. For more information, see [Rate Limits documentation](/api-reference/rate-limits).

#### [Date - e.g., November 10, 2024]

* **Feature Launch**: We've launched [Feature Name] in beta, enabling [capability description]. To enable this feature, use the beta header `feature-name-2024-11-10`. Learn more in our [feature documentation](/docs/feature-name).

* **Model Update**: Description of model or service updates.

* **Console Update**: Description of changes to the CreatorCommerce Console or dashboard.

## Template Notes

When adding new release notes:

1. **Date Format**: Use `#### [Month Day, Year]` format (e.g., `#### January 15, 2025`)
2. **Bullet Points**: Use `*` for each release item
3. **Links**: Include links to relevant documentation using `[link text](/path/to/docs)`
4. **Emphasis**: Use `**bold**` for feature names or important terms
5. **Code**: Use backticks for code elements like endpoint names, headers, or parameters
6. **Beta Features**: Mention beta headers when applicable
7. **Breaking Changes**: Clearly mark breaking changes and provide migration paths
8. **Deprecations**: Include deprecation timelines and replacement options

## Example Entry Structure

```markdown
#### [Date]

* **Feature Name**: Brief description. Learn more in our [documentation](/path/to/docs).

* **API Endpoint**: Description of endpoint changes. See the [API reference](/api-reference/endpoint).

* **Breaking Change**: Description with [migration guide](/path/to/migration).

* **Deprecation**: Description with timeline. See [deprecation docs](/path/to/deprecations).
```
