The Creator Dashboard is where creators manage collaborations, drops, onboarding data, and profile settings. There are two supported paths:

## 1. Customize the Hosted Dashboard with CSS

Use CreatorCommerce's hosted dashboard and theme it with stable selectors when you want a fast launch and minimal maintenance.

This path is best when you want:

- Quick implementation
- Brand alignment through CSS
- No maintenance burden for core dashboard functionality
- CC-managed product updates and feature rollout

Canonical references:

- [Creator Dashboard CSS Overview](/references/dashboard-overview-css)
- [CSS Reference](/references/css-reference)
- [Code Examples & Practical Patterns](/references/css-applied-examples)

## 2. Build a Custom Dashboard via API

Build a completely custom creator dashboard using the CreatorCommerce API and integrate it into Shopify's Customer Account Portal. This approach gives you:

- **Full design control**: Build the UI from scratch using your preferred framework
- **Complete customization**: Match your brand's exact design system
- **Flexible workflows**: Create custom onboarding flows and user journeys
- **Native integration**: Embed within Shopify's Customer Account Portal for a seamless experience

Canonical references:

- [Build Your Own Dashboard](/guides/creator-experience/build-your-own/overview)
- [Authenticate a Custom Creator Dashboard](/guides/creator-experience/build-your-own/auth)
- [Authentication](/guides/authentication)
- [Unified API Reference](/references/unified-api-reference)

## Choose the Right Path

| Need | Best path |
|------|-----------|
| Launch quickly and keep maintenance low | Hosted dashboard + CSS |
| Build custom workflows, layouts, or data flows | Custom dashboard via API |
| Add a few brand touches without rebuilding the product | Hosted dashboard + CSS |
| Unify creator UX with a broader custom customer-account experience | Custom dashboard via API |
