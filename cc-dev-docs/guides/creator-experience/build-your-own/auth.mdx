Partners need to log in to your custom dashboard. Here are the patterns.

## Options

### CC-Issued Tokens

Use CC's auth system, display your UI:

```javascript
// 1. Redirect to CC login
window.location = `https://auth.creatorcommerce.com/login?
  redirect=${encodeURIComponent(window.location.origin + '/callback')}`;

// 2. Handle callback with token
const token = new URLSearchParams(window.location.search).get('token');

// 3. Use token for API calls
const response = await fetch('https://api.creatorcommerce.com/creators/me', {
  headers: { 'Authorization': `Bearer ${token}` }
});
```

**Pros:** No auth infrastructure to build
**Cons:** Partners see CC login briefly

### SSO from Your System

If partners already log into your system:

```javascript
// 1. Partner logs into your system

// 2. Generate SSO token for CC
const ssoToken = await yourBackend.generateCCSSOToken(partnerId);

// 3. Exchange for CC access token
const ccToken = await cc.auth.exchangeSSO(ssoToken);

// 4. Use CC token for API calls
```

**Pros:** Seamless single sign-on
**Cons:** SSO setup required

### Magic Links

Email-based authentication:

```javascript
// 1. Partner requests access
await cc.auth.sendMagicLink(email, {
  redirectTo: 'https://partners.yourbrand.com/dashboard'
});

// 2. Partner clicks link in email

// 3. Your app receives token in URL
const token = new URLSearchParams(window.location.search).get('token');
```

**Pros:** No passwords
**Cons:** Requires email access

### Your Own Auth + CC API Keys

Manage auth yourself, use API keys for CC:

```javascript
// 1. Partner logs into your system (your auth)

// 2. Your backend uses CC API key
const partner = await cc.partners.get(partnerId, {
  apiKey: process.env.CC_API_KEY
});

// 3. Return data to your frontend
```

**Pros:** Full control
**Cons:** More infrastructure

## Token Management

### Storage

| Method | Lifespan | Security |
|--------|----------|----------|
| Memory only | Session | Highest |
| sessionStorage | Tab | High |
| localStorage | Persistent | Medium |
| Cookie (httpOnly) | Persistent | High |

### Refresh

CC tokens expire. Handle refresh:

```javascript
async function callCCAPI(endpoint) {
  let response = await fetch(endpoint, {
    headers: { 'Authorization': `Bearer ${getToken()}` }
  });
  
  if (response.status === 401) {
    await refreshToken();
    response = await fetch(endpoint, {
      headers: { 'Authorization': `Bearer ${getToken()}` }
    });
  }
  
  return response.json();
}
```
