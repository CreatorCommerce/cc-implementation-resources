Collection forms are the backbone of creator product curation. A collection form combines the products a creator curated together with extra questions you want answered about that collection or drop.

Use collection-level questions for data that belongs to the whole curated collection, such as the collection title, description, theme, seasonal angle, featured media, or campaign story. Use product forms for per-product notes, ratings, reviews, UGC, or "why I love this" copy.

Collection forms can appear inside onboarding forms or custom forms. They can also render product forms when the creator adds products.

## Use Cases

| Form | Collects | When to Use |
|------|----------|-------------|
| First drop | Products, collection title, description | Initial onboarding |
| Seasonal picks | Holiday picks, summer favorites, campaign theme | Seasonal marketing |
| Gift guide | Curated products, audience angle, hero copy | Campaign-specific storefronts |
| Content collection | Featured media, collection-level UGC, usage rights | Content campaigns |

## Form Types

### Bio Collection

```
Fields:
- Short bio (2-3 sentences)
- Long bio (paragraph)
- Tagline
- "Why I love [Brand]"
```

### Social Content

```
Fields:
- Image upload
- Video URL
- Caption/description
- Usage rights checkbox
```

### Testimonial

```
Fields:
- Product select (dropdown)
- Rating (1-5)
- Written testimonial
- Photo with product (optional)
```

## Triggering Collection

### Email Campaigns

Send collection requests via email:
- Subject: "Share your summer favorites"
- Magic link to form
- Deadline for submissions

### Dashboard Prompts

Show in-dashboard prompts:
- "Your bio is empty—add one now"
- "We're collecting holiday content"

### Automated Triggers

Workflow-triggered collection:
- 30 days since last update
- New product launch
- Milestone reached

## Progressive Collection

Don't ask for everything at once:

```
Week 1: Onboarding basics
Week 2: Bio and photo
Week 4: Product picks
Week 8: Testimonial request
```

## Product Visibility

Products that appear in search and default selection are controlled in CreatorCommerce settings:

- Use **Exclude Products with Tags** to omit products from creator search and product selection.
- Use **Recommended Collection ID** to show a default product collection in forms.

See [Initial Settings Configuration](/guides/initial-settings#creator-settings) for the setup details.

## Content Moderation

Options for reviewing submitted content:
- **Auto-publish:** Goes live immediately
- **Queue for review:** Admin approval required
- **AI screening:** Automated quality check
