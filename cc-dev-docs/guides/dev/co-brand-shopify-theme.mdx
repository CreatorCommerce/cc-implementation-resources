## Overview

Co-branding your Shopify theme means making sections and templates that dynamically personalize based on creator context. This guide covers:

- Choosing the right build pattern for your goal
- Accessing creator data in Liquid
- Building co-branded sections
- Implementing CSS variables for theming
- Handling fallbacks gracefully

---

## Choosing Your Build Pattern

Before writing code, identify which pattern fits your goal. Each has different data access, complexity, and use case trade-offs.

| Goal | Pattern | Complexity | Key Decision |
|------|---------|------------|-------------|
| **Create a new co-branded section** | Build from scratch with creator resolution, dynamic text, and fallbacks | Medium | Use `metaobject` on landing pages, cart attributes elsewhere |
| **Convert an existing section to support drops** | Duplicate the section, add drops detection, keep original as fallback | Medium-High | Never modify the original file — create a `cc-` prefixed copy |
| **Add creator context to existing pages** | Use cart attribute resolution + conditional rendering | Low-Medium | Wrap all creator elements in `{% if cc_creator %}` guards |
| **Build a creator landing page** | Create a metaobject page template with CC sections | Low | Use `metaobject` directly — no cart attribute resolution needed |
| **Build a creator directory** | Iterate the full metaobject dataset (not cart context) | Medium | Always filter by public status, cap iteration for performance |
| **Hide/suppress UI during creator funnels** | Add Liquid + JS near `</body>` with MutationObserver | Low | Check both `cc_creator` exists AND has discount data |

### How to Decide

**"I want a dedicated page for each creator"** → Build a [landing page](/guides/storefronts/getting-started-landing-pages) using a metaobject page template

**"I want to personalize my existing homepage when a creator link is clicked"** → Add creator context to existing pages using cart attribute resolution

**"I have a product grid or carousel I want to show drops in"** → [Convert the existing section](/guides/storefronts/migrate-existing-sections) to support drops with dual-mode rendering

**"I want a new hero, testimonial, or banner section with creator data"** → Build a new co-branded section from scratch (see below)

**"I want shoppers to browse and discover creators"** → Build a [creator directory](/guides/storefronts/creator-directory) that iterates the full metaobject dataset

## Creator Data Access Patterns

### On Metaobject Pages

When on a creator's dedicated page (metaobject template):

```liquid
{% comment %} Creator data is available directly {% endcomment %}
{% assign cc_creator = metaobject %}

{{ cc_creator.cc-creator-first-name }}
{{ cc_creator.cc-creator-shop-description }}
{{ cc_creator.data.value.cc-creator-profile-picture }}
```

### On All Other Pages

Resolve creator from cart attributes:

```liquid
{% liquid
  assign cc_handle = cart.attributes['cc-creator-handle']
  assign cc_creator = metaobjects.creator[cc_handle]
%}

{% if cc_creator %}
  {{ cc_creator.cc-creator-first-name }}
{% endif %}
```

### Universal Pattern

Handle both contexts:

```liquid
{% liquid
  if metaobject
    assign cc_creator = metaobject
  else
    assign cc_handle = cart.attributes['cc-creator-handle']
    assign cc_creator = metaobjects.creator[cc_handle]
  endif
  
  assign has_creator = cc_creator != blank
%}
```

## Building a Co-branded Section

### Basic Section Structure

```liquid
{% comment %} sections/cc-hero.liquid {% endcomment %}

{% liquid
  if metaobject
    assign cc_creator = metaobject
  else
    assign cc_handle = cart.attributes['cc-creator-handle']
    assign cc_creator = metaobjects.creator[cc_handle]
  endif
  
  assign has_creator = cc_creator != blank
  assign creator_name = cc_creator.cc-creator-first-name | default: section.settings.fallback_heading
  assign creator_bio = cc_creator.cc-creator-shop-description | default: section.settings.fallback_description
  assign creator_image = cc_creator.data.value.cc-creator-profile-picture
  assign discount_code = cc_creator.cc-collab-discount-code
%}

<section class="cc-hero">
  {% if has_creator %}
    <div class="cc-hero__creator">
      {% if creator_image %}
        <img 
          src="{{ creator_image | image_url: width: 400 }}" 
          alt="{{ creator_name | escape }}"
          loading="lazy"
          class="cc-hero__avatar"
        >
      {% endif %}
      
      <h1 class="cc-hero__title">
        {{ creator_name }}'s Picks
      </h1>
      
      <p class="cc-hero__bio">
        {{ creator_bio | escape }}
      </p>
      
      {% if discount_code %}
        <div class="cc-hero__discount">
          Use code <strong>{{ discount_code }}</strong> for 
          {{ cc_creator.cc-collab-discount-amount }}% off
        </div>
      {% endif %}
    </div>
  {% else %}
    <div class="cc-hero__fallback">
      <h1>{{ section.settings.fallback_heading }}</h1>
      <p>{{ section.settings.fallback_description }}</p>
    </div>
  {% endif %}
</section>

{% schema %}
{
  "name": "CC Hero",
  "settings": [
    {
      "type": "text",
      "id": "fallback_heading",
      "label": "Fallback Heading",
      "default": "Featured Products"
    },
    {
      "type": "textarea",
      "id": "fallback_description",
      "label": "Fallback Description",
      "default": "Shop our curated collection"
    }
  ],
  "presets": [
    {
      "name": "CC Hero"
    }
  ]
}
{% endschema %}
```

## CSS Variables for Dynamic Theming

### Injecting Creator Colors

```liquid
{% if cc_creator and cc_creator.cc-creator-shop-theme-primary-color %}
  <style>
    :root {
      --cc-primary-color: {{ cc_creator.cc-creator-shop-theme-primary-color | default: '#000000' }};
      --cc-secondary-color: {{ cc_creator.cc-creator-shop-theme-secondary-color | default: '#666666' }};
      --cc-accent-color: {{ cc_creator.cc-creator-shop-theme-accent-color | default: '#ff5733' }};
    }
  </style>
{% endif %}
```

### Using CSS Variables

```css
/* In your theme CSS */
.cc-hero {
  background-color: var(--cc-primary-color, #ffffff);
}

.cc-hero__title {
  color: var(--cc-primary-color, #000000);
}

.cc-hero__cta {
  background-color: var(--cc-accent-color, #ff5733);
  color: white;
}

.cc-hero__discount {
  border-color: var(--cc-accent-color, #ff5733);
}
```

### Contrast Color Calculation

```liquid
{% comment %} snippets/cc-calculate-contrast.liquid {% endcomment %}

{% liquid
  assign bg_color = cc_creator.cc-creator-shop-theme-primary-color | default: '#ffffff'
  assign bg_color_clean = bg_color | remove: '#'
  
  assign r = bg_color_clean | slice: 0, 2 | hex_to_decimal
  assign g = bg_color_clean | slice: 2, 2 | hex_to_decimal
  assign b = bg_color_clean | slice: 4, 2 | hex_to_decimal
  
  assign luminance = r | times: 299 | plus: g | times: 587 | plus: b | times: 114 | divided_by: 1000
  
  if luminance > 128
    assign text_color = '#000000'
  else
    assign text_color = '#ffffff'
  endif
%}

<style>
  .cc-section {
    background-color: {{ bg_color }};
    color: {{ text_color }};
  }
</style>
```

## Product Grid with Creator Drops

### Displaying Creator's Product Picks

```liquid
{% comment %} sections/cc-drops-grid.liquid {% endcomment %}

{% liquid
  if metaobject
    assign cc_creator = metaobject
  else
    assign cc_handle = cart.attributes['cc-creator-handle']
    assign cc_creator = metaobjects.creator[cc_handle]
  endif
%}

{% if cc_creator and cc_creator.cc-creator-drops.value %}
  {% assign drops_data = cc_creator.data.value.cc-creator-drops %}
  
  <section class="cc-drops-grid">
    <h2>{{ cc_creator.cc-creator-first-name }}'s Picks</h2>
    
    {% for drop in drops_data %}
      {% assign drop_collection_id = drop['cc-creator-drop-collection-id'] %}
      
      {% for collection in collections %}
        {% comment %} Convert to string for comparison {% endcomment %}
        {% if collection.id | append: '' == drop_collection_id %}
          <div class="cc-drop">
            <h3>{{ drop['cc-creator-drop-name'] | default: collection.title }}</h3>
            
            <div class="cc-drop__products">
              {% for product in collection.products limit: 4 %}
                {% render 'cc-product-card', product: product, cc_creator: cc_creator %}
              {% endfor %}
            </div>
          </div>
        {% endif %}
      {% endfor %}
    {% endfor %}
  </section>
{% endif %}
```

### Product Card with Creator Context

```liquid
{% comment %} snippets/cc-product-card.liquid {% endcomment %}

<div class="cc-product-card">
  <a href="{{ product.url }}">
    <img 
      src="{{ product.featured_image | image_url: width: 300 }}" 
      alt="{{ product.title | escape }}"
      loading="lazy"
    >
  </a>
  
  <h4>{{ product.title }}</h4>
  
  {% if cc_creator and cc_creator.cc-collab-discount-amount %}
    {% assign discount = cc_creator.cc-collab-discount-amount | times: 0.01 %}
    {% assign discounted_price = product.price | times: 1 | minus: product.price | times: discount %}
    
    <div class="cc-product-card__price">
      <span class="cc-product-card__original-price">
        {{ product.price | money }}
      </span>
      <span class="cc-product-card__sale-price">
        {{ discounted_price | money }}
      </span>
    </div>
  {% else %}
    <div class="cc-product-card__price">
      {{ product.price | money }}
    </div>
  {% endif %}
  
  <button 
    class="cc-product-card__add-to-cart"
    data-product-id="{{ product.variants.first.id }}"
  >
    Add to Cart
  </button>
</div>
```

## Accessing Custom Field Data

### Custom Fields in Sections

Custom fields collected via [CC's form system](/guides/creator-experience/forms/overview) are stored in the creator metaobject and accessible via Liquid. They exist at three levels:

```liquid
{% comment %} Collab-level custom field (one per partnership) {% endcomment %}
{% assign creator_specialty = cc_creator.data.value['cc-creator-specialty'] %}

{% comment %} Drop-level custom field (within a drop loop) {% endcomment %}
{% for drop in cc_creator.data.value['cc-creator-drops'] %}
  {% assign drop_tagline = drop['cc-drop-tagline'] %}
{% endfor %}

{% comment %} Product-level custom field (within a product loop inside a drop) {% endcomment %}
{% for product_item in drop['cc-creator-drop-products'] %}
  {% assign love_note = product_item['cc-product-love-note'] %}
{% endfor %}
```

### Conditional Rendering with Custom Fields

```liquid
{% liquid
  assign creator_specialty = cc_creator.data.value['cc-creator-specialty'] | default: ''
  assign creator_tagline = cc_creator.data.value['cc-creator-tagline'] | default: ''
%}

{% if creator_specialty != blank %}
  <p class="cc-specialty">{{ creator_specialty | escape }}</p>
{% endif %}

{% if creator_tagline != blank %}
  <h2 class="cc-tagline">{{ creator_tagline | escape }}</h2>
{% endif %}
```

<Info>
Custom field names are defined when brands set up [custom fields](/references/custom-fields-reference) in the CC dashboard. Always check the actual field names with the brand and add null guards — custom fields are optional and may not be populated for every creator.
</Info>

## Setting Cart Attributes

### On Landing Page Load

```liquid
{% comment %} In your theme.liquid or section {% endcomment %}

{% if request.path contains '/creators/' or request.path contains '/pages/creators/' %}
  <script>
    (function() {
      const creatorHandle = '{{ cc_creator.handle | escape }}';
      const discountCode = '{{ cc_creator.cc-collab-discount-code | escape }}';
      
      // Set cart attributes
      fetch('/cart/update.js', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          attributes: {
            'cc-creator-handle': creatorHandle,
            'cc-discount-code': discountCode,
            'cc-landing-time': new Date().toISOString()
          }
        })
      });
    })();
  </script>
{% endif %}
```

### Hidden Form Fields

```liquid
{% comment %} In your add-to-cart form {% endcomment %}

{% if cc_creator %}
  <input type="hidden" name="attributes[cc-creator-handle]" value="{{ cc_creator.handle }}">
  <input type="hidden" name="attributes[cc-discount-code]" value="{{ cc_creator.cc-collab-discount-code }}">
{% endif %}
```

## Dynamic Text Replacement

### Token Replacement Snippet

```liquid
{% comment %} snippets/cc-dynamic-text.liquid {% endcomment %}
{% comment %} Usage: {% render 'cc-dynamic-text', text: section.settings.heading, cc_creator: cc_creator %} {% endcomment %}

{% liquid
  assign output = text
  
  if cc_creator
    assign output = output | replace: '[cc-creator-first-name]', cc_creator.cc-creator-first-name
    assign output = output | replace: '[cc-creator-last-name]', cc_creator.cc-creator-last-name
    assign output = output | replace: '[cc-discount-code]', cc_creator.cc-collab-discount-code
    assign output = output | replace: '[cc-discount-amount]', cc_creator.cc-collab-discount-amount
  else
    assign output = output | replace: '[cc-creator-first-name]', 'our expert'
    assign output = output | replace: '[cc-creator-last-name]', ''
    assign output = output | replace: '[cc-discount-code]', ''
    assign output = output | replace: '[cc-discount-amount]', ''
  endif
%}

{{ output }}
```

### Usage in Sections

```liquid
<h1>
  {% render 'cc-dynamic-text', 
    text: section.settings.heading, 
    cc_creator: cc_creator 
  %}
</h1>

{% comment %} Setting value: "Shop [cc-creator-first-name]'s favorites" {% endcomment %}
{% comment %} Output: "Shop Sarah's favorites" {% endcomment %}
```

## Directory Architecture Note

Creator directories are architecturally different from co-branded sections. They iterate the **full metaobject dataset** rather than reading a single creator from cart context. Key principles:

- **No cart attribute resolution** — directories don't use `cart.attributes['cc-creator-handle']`
- **Full dataset iteration** — loop `metaobjects.creator` with public status filtering
- **Performance caps** — always use `break` to limit iterations (Liquid loops through every entry otherwise)
- **Optional: exclude current creator** — on landing pages, hide the active creator from "similar creators" sections

See [Build a Creator Directory](/guides/storefronts/creator-directory) for full implementation patterns.
